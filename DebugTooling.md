# Debug Tooling

## Overview

Two debug outputs help during development:

1. **Symbol files** (`.sym`) — map label names to ROM addresses, loaded by emulators and
   debuggers so you see `main:` instead of `0x0150` in the disassembly pane.
2. **Annotated disassembly** — a human-readable dump of ROM bytes with label names and
   decoded mnemonics; useful for verifying that the assembler emitted the right code.

Both are generated from the same label map that the linker already builds during assembly.

---

## Symbol File Format

Emulicious (and the WLA-DX / NO$GMB family of debuggers) accepts `.sym` files in the following
format:

```
; Game Gear symbol file
[labels]
00:0000 reset
00:0038 vblankISR
00:0066 nmiISR
00:0100 main
00:0150 gameLoop
00:01A0 updatePlayer
...
```

Each line: `BB:AAAA name` where `BB` is the ROM bank (always `00` for a ≤32 KB ROM) and
`AAAA` is the 16-bit address.

The symbol file must have the same base name as the ROM:

```
demo.gg    → demo.sym
```

Emulicious loads it automatically when you open the ROM.

## Generating Symbol Files

The linker already computes a `Map Label Word16` as part of two-pass assembly.  Exporting
it is a small addition to `ROM/Builder.hs`.

```haskell
import qualified Data.Map.Strict as Map
import Data.Text (Text)
import qualified Data.Text as T
import qualified Data.Text.IO as TIO

-- | Render a label map as a Emulicious-compatible .sym file body.
renderSymFile :: Map.Map Label Word16 -> Text
renderSymFile labels =
  "[labels]\n" <> foldMap renderEntry (Map.toAscList labels)
  where
    renderEntry (Label name, addr) =
      T.pack (printf "00:%04X " addr) <> name <> "\n"

-- | Write a .sym file alongside the ROM.
writeSymFile :: FilePath -> Map.Map Label Word16 -> IO ()
writeSymFile romPath labels =
  let symPath = replaceExtension romPath ".sym"
  in  TIO.writeFile symPath (renderSymFile labels)
```

This requires surfacing the label map from `runLinker` — currently it returns only
`Either LinkerError ByteString`.  The suggested change:

```haskell
data AssemblyResult = AssemblyResult
  { romBytes :: ByteString
  , symbolMap :: Map.Map Label Word16
  }

assemble :: ROMConfig -> Asm () -> Either AssemblerError AssemblyResult
```

## Annotated Disassembly

An annotated disassembly interleaves label markers with decoded instruction lines:

```
00:0000  main:
00:0000  F3          DI
00:0001  31 FF DF    LD SP, $DFFF
00:0004  CD 50 01    CALL $0150       ; updatePlayer
00:0007  18 F7       JR $0000         ; gameLoop
```

The `; label` comment on indirect jumps/calls requires cross-referencing the target address
against the symbol map.

### Generating Disassembly

A disassembler walks the `ByteString` byte-by-byte, decoding each instruction from the Z80
encoding tables (the same tables used in `Encoder.hs`, inverted).  Label insertion comes from
the symbol map.

```haskell
-- | Render annotated disassembly as Text.
-- @bytes@ = ROM content, @syms@ = address→name map.
disassemble :: ByteString -> Map.Map Word16 Text -> Text
disassemble bytes syms = T.unlines (go 0)
  where
    go pc
      | pc >= BS.length bytes = []
      | otherwise =
          let (mnemonic, len) = decodeAt bytes pc
              labelLine = case Map.lookup (fromIntegral pc) syms of
                            Just name -> [T.pack (printf "00:%04X  %s:" pc name)]
                            Nothing   -> []
              hexBytes = T.pack (unwords (map (printf "%02X") (BS.unpack (BS.take len (BS.drop pc bytes)))))
              instrLine = T.pack (printf "00:%04X  %-12s %s" pc hexBytes mnemonic)
          in labelLine ++ [instrLine] ++ go (pc + len)
```

`decodeAt` is the inversion of `Encoder.hs` — a separate module `Z80/Disassembler.hs`.

## Emulicious Integration

1. Open Emulicious → File → Open ROM → `demo.gg`
2. The `.sym` file in the same directory is loaded automatically.
3. In the debugger pane, breakpoints can be set by label name.
4. The memory viewer shows label annotations in the address column.

The DAP-based test infrastructure in `test/Emulicious/` already launches Emulicious
programmatically; `Runner.hs` could be extended to pass `--symfile` or set a breakpoint at a
named label rather than a hard-coded address.

## Notes

- **Internal labels** (prefixed `_`, generated by `freshLabel`) should be filtered out of the
  `.sym` file — they clutter the debugger with thousands of loop-top and VBlank labels.
  Only export labels created via `defineLabel`.
- **Bank-qualified addresses**: Once bank switching is added, symbol addresses must include the
  bank number.  Reserve the `BB:AAAA` format now even though `BB` is always `00` for ≤32 KB
  ROMs.
- **Line numbers**: If the DSL is ever extended to track source positions (e.g., via a
  `ReaderT SourcePos` wrapper), the `.sym` format supports an optional `[source]` section that
  maps addresses to `file:line` pairs — useful for a future source-level debugger.
